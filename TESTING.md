# 团团 · 测试与上线指南

## 三个环境（靠 URL 参数自动切换，见 `js/config.js`）

| 环境 | 怎么进 | 连哪个后端 | 用途 |
|---|---|---|---|
| **本地演示 demo** | 网址加 `?demo` | 不连后端（localStorage） | 最快迭代、测 UI/交互。**0 延迟**（测不到真实快慢） |
| **测试 test** | 网址加 `?test` | 测试库（独立 Sheet/GAS） | 测真实后端 + 真实延迟，**数据不污染线上** |
| **正式 prod** | 不加参数 | 正式后端（线上） | 真实用户用的 |

非正式环境页面顶部会有 **🧪 标记**，避免把测试当成线上误操作。

## 一、本地测试（最常用）

**双击 `test-local.cmd`** —— 自动起本地服务器并打开三端（demo 模式）。
关闭弹出的「TuanTuan server」窗口即停止。

> 手动方式：在本目录运行 `python -m http.server 8777`，浏览器开
> `http://localhost:8777/index.html?demo`（客户）、`merchant.html?demo`（商家）、`admin.html?demo`（管理）。

## 二、搭建「测试后端」（只需做一次，用于 `?test` 测真延迟、不脏线上）

后端是独立 Apps Script，会**自动建自己的 Sheet**，所以复制脚本即可得到隔离的测试库：

1. 打开正式后端的 Apps Script 项目 → 右上「⋮」→ **创建副本**（命名如「团团后端-测试」）。
2. 副本里：**部署 → 新建部署 → 类型选「网页应用」**，执行身份 = 我，访问权限 = 任何人 → **部署**，复制 `/exec` 网址。
3. 把该网址填进 `js/config.js` 的 `TEST_API`。
4. 用 `admin.html?test` 登录管理员 → 用「测试工具」里的 **重新播种(resetSeedData)** 给测试库灌入种子数据。
   （测试库的管理员密码、TOKEN 与线上独立，按需在副本的脚本属性里设。）

之后任意页面加 `?test` 就走测试库：真实后端、真实延迟、数据与线上完全隔离。

## 三、上线与回滚

- **前端上线**：改动推到 `master` → 自动部署。低风险、可回滚（`git revert` 后再推）。
- **建议流程**：本地 `?demo` 快速改 → 上线前 `?test` 对真实后端验收（含延迟）→ 没问题再推 `master`。
- **后端上线**：先在「测试副本」改+部署、用 `?test` 验 → 再把同样的代码贴回正式项目并**重新部署**。
- 线上数据始终在自己的库里，测试永远打测试库，互不影响。
