## 用户接受的「证据」存哪

### 客户

客户首次填资料时勾选「同意条款」checkbox → 同时保存：
- **localStorage** `tt_tos_accepted = "1"` （客户端记忆，避免再问）
- **localStorage** `tt_tos_accepted_at = "2026-05-31T12:34:56Z"`（时间戳）
- **localStorage** `tt_tos_version = "v1.0"` （记录接受的版本）

**beta 阶段「证据」存本地是可接受的**。一旦发生纠纷：
- 客户继续使用 = 同意条款的事实推定
- 法律上「点击同意」+ 「使用记录」已构成 contract

> ⚠️ 商业化规模时升级：把同意记录写到后端 `ConsentLogs` 表，留 admin 可查的审计日志。

### 商家

商家在开通时：
- **离线**：让商家 WhatsApp 回复「我同意 v1.0 商家协议」→ 截图存档
- **半自动**：你在 admin 后台「新增商家」时打个勾「商家已同意 v1.0 协议」→ 写入 Vendors.tosVersion 字段（暂未实现）

## 法律文档 vs 实际执行

很多人误以为「写了 T&C 就能为所欲为」。**不是这样**：

✅ **T&C 能做的**：
- 把可预见的纠纷责任分清楚
- 给违规用户/商家一个驱逐的法律依据
- 提供一个「我们认真做事」的形象

❌ **T&C 不能做的**：
- 让你免除「重大过失」的责任
- 推翻消费者保护法（如食品安全）
- 替代实际的商家筛选 + 客服

**所以**：
- 上架商家前**实地（或视频）看一下店面**，不是阿猫阿狗都收
- 客户投诉**24h 内必回**（哪怕只是说「我了解了正在处理」）
- 重大事故（如食物中毒）**立刻下架该商家** + 配合客户走正常途径

## 适合 beta 但不适合规模化的 5 个地方

这 3 份文档在以下场景**不够**，建议届时升级：

1. **平台开始抽成或代收代付** → 涉及金融服务法律，必须找律师
2. **跨州运营**（不止 UTM/UKM）→ 增加跨州配送和税务条款
3. **雇佣员工**（你之外有运营/客服）→ 加员工保密条款
4. **接受外部投资** → 投资方会要求律师从头写一遍
5. **拓展到大马境外** → 各国法律差异巨大

每一条触发时，**主动找律师**，别拖。
