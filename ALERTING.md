# 自动告警系统

> **每小时自动检查系统健康，异常时主动 push 到你手机锁屏**。半夜出事你也能立刻知道。

完全 0 成本 · 完全 0 维护 · 跑在 Cloudflare Worker Cron 上。

---

## 一、它做什么

每个整点（00:00, 01:00, 02:00 ...）：

```
CF Worker Cron 触发
    │
    └─→ POST /exec systemSelfCheck (workerSecret 鉴权)
            │
            └─→ GAS 内部跑 5 项检查：
                ✅ Sheet 7 张表都能访问
                ✅ GAS 配额占比（>60% 警告 / >80% 严重）
                ✅ Sheet cells 占比（>80% 严重）
                ✅ 订单 / 商家 / 订阅 数据量
                ✅ 失效订阅堆积（failCount≥3 多于 5 个）
            │
            └─→ 发现异常？
                ├─ 是 → push 给所有 role='admin' 订阅设备
                │       （标题 ⚠ 团团系统异常 + 前 3 项具体描述）
                │       小时桶 tag 去重：同 1 小时不会刷屏
                │
                └─ 否 → 静默（不打扰）
```

## 二、首次设置（5 分钟）

### Step 1 · Worker 重新部署（带 Cron）

```powershell
cd D:\MyProject\order-delivery\worker
wrangler deploy
```

部署成功后，输出会带：
```
Cron Triggers:
  - "0 * * * *" (every hour)
```

如果没看到这行，检查 `wrangler.toml` 是否有 `[triggers]` 段（应该有）。

### Step 2 · GAS 重新部署 Code.gs

Apps Script 编辑器 → 部署 → 管理部署 → 修订版 → 新版本 → 描述写 `Health check v1`

### Step 3 · admin 订阅推送（关键！）

**这步不做的话，告警发不出来**：

1. 浏览器开 `https://你的域/admin.html`
2. 用 admin 账号登录
3. 登录后会弹「订阅订单状态推送」横幅
4. 点 **「允许通知」** → 浏览器弹系统权限 → 选「允许」
5. 完成后，你的浏览器/手机就被记录为 admin 推送目标

**重复同样步骤在你的手机浏览器** 也来一遍 —— 这样手机锁屏也能收到告警。

> **Tip**：手机想接锁屏告警，需要先「加到主屏」装成 PWA。Safari 用户进 admin.html 后看分享按钮 → 「添加到主屏幕」。

### Step 4 · 立即测试一次

1. 浏览器开 admin.html → 🧪 测试工具 tab
2. 点 **「🤖 Worker 健康检查 → 立即跑一次」**
3. 看到 `状态: 一切正常 ✓` = 全链路通了

## 三、收到告警怎么处理

### 例：告警弹出「⚠ 团团系统异常 (2 项)」

锁屏推送内容大概像：
```
⚠ 团团系统异常 (2 项)
GAS 配额 75% (警戒线)；3 个订阅连续失败...
```

**响应剧本**：

1. **点通知** → 自动跳 admin.html
2. 进 🧪 测试工具 → 「🤖 Worker 健康检查」 → 看完整问题列表
3. 对症处理：

| 告警 | 处理 |
|---|---|
| `GAS 配额 60-80%` | 不紧急；考虑启用 ARCHITECTURE.md 阶段 2 |
| `GAS 配额 > 80%` | **当天剩余时间随时可能崩**。立即关掉非必要查询、删历史 isTest 数据 |
| `Sheet cells > 80%` | 接近 10M 上限。归档老订单 / 删 SystemLogs 旧行 |
| `表 X 异常: ...` | Sheet 可能被手动改坏了。先看 Sheet → 版本历史恢复 |
| `N 个订阅连续失败` | 客户卸载浏览器等。不紧急，可在 admin 清理或等自动 5 次失败清除 |
| `selfCheck 顶层异常` | Code.gs 出 bug 或部署不一致。先 GAS → 管理部署 → 选上一版本 |

## 四、告警的「不」清单（避免被噪音淹没）

- ❌ **不在每次失败时都推**：1 小时只推一次（同样问题）
- ❌ **不在恢复时也推**：避免「恢复通知」也成为噪音（你打开 admin 一看就知道好了）
- ❌ **不在 admin 没订阅时报错**：先帮记录到 SystemLogs，下次 admin 进 admin.html 自然能看到
- ❌ **不替代 admin 监控面板**：告警只在「主动出事」时叫你；日常你还是该进 admin 看监控

## 五、配额开销（再次确认 0 成本）

| 资源 | 用量 |
|---|---|
| CF Workers 调用 | 24 次/天（Cron）= **0.024% of 100K 配额** |
| CF Workers CPU | 每次 ~3ms = 完全可以忽略 |
| GAS 执行时长 | 每次 ~500ms = 12s/天 = **0.2% of 90 min 配额** |
| Web Push 推送 | 异常时才推 = 0-3 次/天平均 |

**实际成本：完全 0**。这就是 Cron Triggers 留在免费层的最大价值。

## 六、关闭告警（如果你嫌烦）

3 种方法：

1. **临时关**：admin 进浏览器设置 → 通知 → 拒绝 团团 站点
2. **彻底关**：在 admin → 🧪 测试工具运行下面这段（清掉 admin 订阅）：
   ```javascript
   // TODO: 加一个「清掉我所有 admin 订阅」按钮
   ```
3. **关 Cron 不发推**：编辑 `worker/wrangler.toml` 删 `[triggers]` 段，`wrangler deploy`
